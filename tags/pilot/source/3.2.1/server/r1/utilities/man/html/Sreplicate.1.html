NAME

Sreplicate - replicates an SRB object  

SYNOPSIS

Sreplicate [-n replicaNum] [-prv] [-S resourceName] [-P pathName] srbObj|collection ...  

DESCRIPTION

Sreplicate makes one more copy of srbObj or collection. The replica is also registered in MCAT.

The user should have at least 'read' access permission for srbObj or collection.

For replicating collections, the recursive option (r) must be set.

If given, the object is stored in resourceName. Otherwise the object is stored in the default resource given by DEFRESOURCE in the user environment file: ~/.srb/.MdasEnv. If the resourceName is a logical resource consisting of multiple resources, only one replica will be made to one of the resources in the group. The cache type resource will be used first before the archival type resources. Unlike the Ssyncd command, an additional copy will be made even though one or more replica already exist in the resource.

If given, the object is stored in pathName. Otherwise the object is stored in the default path for the resource stored in MCAT.

The srbObj or collection arguments can be path names in the SRB collection hierarchy.

 

OPTIONS

-h
display command options
-p
prompts before replicating each object.
-r
recursively replicating the collection.
-v
verbose mode. print out file size and transfer rate.
-n
replicaNum is an integer denoting the COPY of the object to be replicated.
-P
full or relative newpathName to move the object.
-S
new resourcename
