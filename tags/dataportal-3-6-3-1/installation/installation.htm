Installation of the Data Portal using a Tomcat Servlet engine

1)  Install Apache Tomcat servlet engine.  Need version 5.0+.  Download http://jakarta.apache.org/site/binindex.cgi

Tomcat 5.0.X and the Data Portal modules uses java jre version 1.4.x (preferably 1.4.2) and does not run on version 1.5 (or java 5 'tiger') or 1.3.

(NB:  Make sure Environmental variable CATALINA_HOME is set correctly if you are installing a new instance of tomcat)

2)  Install Apache Ant.  Need Version 1.6+.  Download http://ant.apache.org/bindownload.cgi

Copy $CATALINA_HOME/server/lib/catalina-ant.jar into $ANT_HOME/lib where $CATALINA_HOME represents the tomcat root directory and $ANT_HOME represents the ant root directory.

(NB: Red Hat Linux comes preinstalled with ant, usually in /usr/bin/ant.  You need to use apache ant, which you need to have installed.  Either make sure apache ant is in front in you PATH variable or you use the full path when executing ant.  I.e $ANT_HOME/bin/ant)

3) Change the properties for the $DATAPORTAL_HOME/build.properties file.

Where $DATAPORTAL_HOME represents the root directory where you unpacked the data portal file.

server.name=localhost:8080    //server name and port of tomcat instance.  localhost is ok if it is on the same machine

catalina.home=D\:\\tomcat    // location on file system of tomcat instance. (\ is an escape character. So windows needs to escape : and \ within the path. IE c\:\\My Documents\\ etc)

manager.username=tomcat 

manager.password=tomcat

globusKey=C\:\\Documents and Settings\\gjd37\\.globus\\portalkey.pem 

globusCert=C\:\\Documents and Settings\\gjd37\\.globus\\portalcert.pem

 

The last two are the locations of your globus host key and certificate.  These are used to create a proxy that is used to retrieve the user's proxy credential out of a MyProxy server.  Host credentials for Uk e-Science can be requested from Uk eScience CA.

(NB: The Data Portal requires a MyProxy Server to request a user credential to authenticate the user to the Data Portal.  To request a user's credential the Data Portal needs a host public and private key in Globus format to authenticate itself to the MyProxy server, the DN of this host certificate must be allowed to get user's credentials out of the MyProxy server).

manager.password and manager.username are needed to install the Data Portal modules onto tomcat.  Change the file $CATALINA_HOME/conf/tomcat-users.xml.

Insert this line or modify by adding a role manager to the role attribute of the user xml tag highlighted in blue here:

<user username="tomcat" password="tomcat" roles="tomcat , manager"/>

 

Making sure the username and password are matching the ones in the build.properties file.

6) ant install_dp    (NB: All ant commands need to be executed in the $DATAPORTAL_HOME directory)

This installs the UDDI, databases and relevant libraries into your tomcat instance.

(NB: This process installs all the axis and jglobus jar files into $CATALINA_HOME/shared/lib.

It also puts dp_common.jar, hsqldb.jar, mail.jar, activation.jar, jdom.jar and the two files jstl.jar and standard.jar in there)

7) ant start_dbs

This starts up the databases.  These are needed for the UDDI registry and the data portal modules.

(NB: ant stop_dbs

stops the two database instances.

(Extra: To view the data portal database run 'ant run_GUI' which should open a database window to the Data Portal database))

8) startup the tomcat instance.  Goto $CATALINA_HOME/bin and execute startup.bat or startup.sh

9) ant -f installation.xml install

If this fails with a

 taskdef A class needed by class org.apache.catalina.ant.InstallTask cannot be found: org/apache/tools/ant/Task

error then:

i) shutdown tomcat ($CATALINA_HOME/bin/shutdown.sh(bat))

ii) ant -f installation.xml install2

iii) then restart tomcat ($CATALINA_HOME/bin/startup.sh(bat))

This installs all the modules to the tomcat instance.  Once installed these are not needed to be installed again and should be present when you restart tomcat again.

THE DATAPORTAL SHOULD BE INSTALLED ONTO YOUR TOMCAT.

Goto http://localhost:8080/dataportal and log in. 

Goto http://tiber.dl.ac.ul:8080/myproxy for help to upload Globus proxy into a MyProxy Server.

Once logged in you should have no access to facilities because there are no facilities associated.  (See below)

(NB:  Press browser refresh if the is an internal section error thrown)

10)  Once installed and running, to start the Data Portal when everything is shutdown:

i) ant start_dbs

ii) start tomcat instance ($CATALINA_HOME/bin/startup.sh(bat))

 

Configuration of the Data Portal

1) Goto http://localhost:8080/facility and log in as username tomcat password tomcat.

2) Click "Add Facility".  There is a test facility that allows access to test information.

3) Put in the name 'test' for the facility name.

4) Change the values in all the URLs for 'localhost' to 'tiber.dl.ac.uk' and click 'Create Facility'

This creates a test facility for the Data Portal to search for.

5) Goto http://localhost:8080/dataportal and log in again, and you should see a test facility for you to search.

(NB: If you make an error in the configuration of the Data Portal facility, press 'delete' next to the facility and start again)

(NB: Since the metadata is test data, there is no data and therefore the data transfer section will not work correctly)

 

Information

1) Log files are written to the console from the Web Interface.  All other modules and the Web Interface log their file to the $CATALINA_HOME/logs/dataportal.xml file.

This file is not really human readable.  It is in the format of Chainsaw.  To view this information download chainsaw at http://logging.apache.org/log4j/docs/chainsaw.html

2) Remove Data Portal from tomcat in two ways.  The first is manual (the second does it by ant scripts.  This removes all jar files in the $CATALINA_HOME/shared/lib directory that were put there before listed in Installation part 6).  Some of these you might not want to delete.

2.1)

 i) ant -f installation remove

  If that fails remove all the xml file related to the data portal in the $CATALINA_HOME/conf/Catalina/localhost directory.

  These should be dataportal, shopcart, facility, juddi, queryandreply, uddilookup, sessionmanager, authent and datatransfer .xml files

 ii) Remove juddi folder from $CATALINA_HOME/webapps

 iii) Remove new jar files from $CATALINA_HOME/shared/lib folder.  Goto Installation part 6) to find out which jar files are installed

or 2.2)

  i) ant remove_dp

3) The Data Portal can run a server to email an administrator regarding errors and faults.

To run the server type 'ant run_log4jserver'

To configure the server go to dataportal/WebInterface/build/WEB-INF/log4jServer.conf

dataportal.id=test                          //id of the email

mail.host=exchange06.dl.ac.uk               //mail server host name of the

email.address=g.j.drinkwater@dl.ac.uk       //list of addresses to send the email to, separtted by spaces

level=ERROR  // Log4j minimum that the message will get emailed.  Only allows ERROR and FATAL

4) Using different databases than HypersonicSQL for the data portal databases. 

i) To configure different databases rather than using the default HypersonicSQL you need to create the database tables in your new database using the scripts $DATAPORTAL_HOME/sql.  Modifying the three sql scripts for your database, the only problem might be with the LONGVARBINARY which is used in the shoppingcart_hsql.sql file to represent a BLOB, and create the tables in the new database.

ii)  Change the username, password, driverClassName and url in the GlobalNamingResources/ResourceParams/parameter/name tags of section 4's installation within the $CATALINA_HOME/cong/server.xml file.

(NB: Goto Apache Tomcat for help on tomcat database pooling)

5) Using different databases than HypersonicSQL for the UDDI databases. 

i) Goto JUDDI for help installing JUDDI on another database. 

ii) Once installed change the file's properties located in $DATAPORTAL_HOME/dataportal/facility/web/WEB-INF/uddi.properties to the uddi_username, uddi_password, uddi_inquiry and uddi_publish of your JUDDI installation.  The code needs to be recompiled with the command 'and -f installation.xml compile' when tomcat is stopped.
