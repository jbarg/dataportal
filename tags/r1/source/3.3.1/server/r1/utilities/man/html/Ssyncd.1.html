NAME

Ssyncd - synchronizes copies of an SRB object  

SYNOPSIS

Ssyncd [-ar] [-S resourceName] srbObj|collection ...  

DESCRIPTION

Ssyncd synchronizes all copies (replica) of an SRB object with the most recently modified (dirty) copy.

If the resourceName is given using the -S option, in addition to synchronizing to all existing copies, a copy will be made in the specified resource if a copy does not already exist in the resource.

If the resourceName is a logical resource consisting of multiple resources, only one copy will be made to one of the resources in the group. The -a option can be used to make a copy in each resources in the group.

For synchronizing collections, the recursive option (r) must be set.

The srbObj or collection arguments can be path names in the SRB collection hierarchy.

 

OPTIONS

-a
If the target resource is a logical resource consisting of more than one resources, then make a copy in each resources in the group.
-r
recursively synchronizing the collection.
-S
resourceName is the target resourceName.
