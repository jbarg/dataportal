Transfer / Download


Options

Currently items (files and datasets) can only be downloaded and transfered. The Data Portal currenty can retrieve items from SRB, HTTP, GridFTP, GASS and FTP severs and transfer them to GridFTP, GASS and FTP severs or just to download to your machine.
SRB (Storage Resource Broker)

SRB items (Data Sets and Files) can be transfered or downloaded using the Data Portal. Click on the download button and the SRB collection will be tar balled and downloaded to your machine as a single file, or downloaded as a single file if the item is a SRB file.

To transfer the item, either the tar balled collection or file can be transferred to any of GridFTP, GASS and FTP severs. This requires you to put in the correct URL for the destination file. Goto Globus URL for more inforation on the URLs. Once the correct URL has been typed press 'transfer' and wait for the reply. If sucessful, a transfer sucessful page will appear, otherwise an error page will be presented with a error. For more information understanding error messages click here.

Files

Files can be transfered or downloaded using the Data Portal (but not Data Sets of files). Click on the download button and the File will be downloaded to your machine.

The file can be transferred to any of GridFTP, GASS and FTP severs. This requires you to put in the correct URL for the destination file. Goto Globus URL for more inforation on the URLs. Once the correct URL has been typed press 'transfer' and wait for the reply. If sucessful, a transfer sucessful page will appear, otherwise an error page will be presented with a error. For more information understanding error messages click here.

Globus


URL

Globus URLs take the same format as HTTP urls. The normal format is

<protocol>://<machine name>/<file location>
For example, if you wanted to do a 3rd party GridFTP transfer, to a machine www.bbc.co.uk and put in the the /tmp directory as a file called myfile the URL would be:
gsiftp://www.bbc.co.uk//tmp/myfile
If you wanted to do a 3rd party HTTPS (GASS) transfer, to a machine www.bbc.co.uk and put in the the c:\temp directory as a file called myfile the URL would be:
https://www.bbc.co.uk/c:/temp/myfile
A URL :
gsiftp://www.bbc.co.uk/tmp/myfile
would represent a file in the tmp directory of you home directory.

Globus Exceptions

Globus Exceptions can be very hard to read and understand. Here is some examples and there meanings

Error: Missing '[protocol]://'     Means that either you have not put a protocol into the URL or the protocol that you have put in is invalid (spelt incorrectly).

Error: UrlCopy transfer failed. [Root error message: escy.dl.ac.uk]     Means that the host is unknown to the system.

Error: UrlCopy transfer failed. [Root error message: Connection refused]     Means that either the machine is down or the globus server is down.
