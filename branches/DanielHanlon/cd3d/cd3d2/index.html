What is it?

 
CrystalDisplay3D is a small application written in Java for the visualization and manipulation of data files related to crystalline structure and electron charge density.

Amongst other things, Java is a platform independent language so CrystalDisplay3D will run on any platform supporting the required Java implementation.

What do I need to run it?

This application has been designed to be based on a web-page and run from there on whichever machine the user happens to have. The only caveat to this, is that the platform must support Java 2 and the standard 3D extension library known as Java3D.

Java3D provides a complete API for developing complex three dimensional scenes in the Java language. Due to the nature of the calculations involved in 3D displays it was decided that the Java3D should be implemented by providing an interface to an underlying platform dependent 3D rendering engine.

Java3D effectively extends the procedures of the standard virtual machine, when dealing with functionality that is intrinsically hardware dependent, to the domain of 3D graphics. By doing so, the programmer is able to work with a standard interface which additionally provides an extra layer of abstraction from the rendering engine to allow scenes to be programmed with a higher level description. The Java3D run-time environment automatically makes use of the 3D processing capabilities present on the client machine. A graphics card that is capable of intercepting OpenGL calls to provide hardware acceleration is automatically employed by Java3D as on that machine the OpenGL version should be installed and the run-time environment would itself be making the calls to OpenGL.

On the Windows/x86 platform, neither of the most popular web-browsers ( Netscape & Internet Explorer ) provide a complete implementation of the Java 2 standard so it is necessary to use the Java Plug-In which may be downloaded at no charge. The following steps are recommended to provide support for CrystalDisplay3D.

Installation Instructions.

Windows/Solaris
    • Download the Java plug-in version 1.2.2 from here.
        • This is the latest release version and it fixes some bugs which greatly complicated the installation process. It is currently, however, only available following free registration with the Java Developers Connection (JDC).
    • Download the Java3D run-time environment from here.
        • For Windows versions are available which use either OpenGL or DirectX as the native rendering engine. The OpenGL version is a later release and should be used unless hardware acceleration for DirectX is present on your machine.
    • Install the Java plug-in.
    • Install the Java 3D run-time environment into the directory within the plug-in.
        • e.g., if the plug-in has been installed to
                C:\Program Files\JavaSoft\JRE\1.2
        this directory should also be used as the installation path for Java3D.
    • Point your browser at the examples web-page.
    • The plug-in will request privileges to be granted to the applet - this is to allow it to utilise the native code involved with the 3D rendering. Agreeing to this will allow the applet to proceed.
Other operating systems.
 
The status of ports to other platforms is continuously changing but implementations of Java 2 (also know as 1.2) and the Java 3D runtime environment are being produced for most environments. In particular first release versions of both pieces of software for Linux are available from http://www.blackdown.org and SGI has release versions of Java 2 and has recently announced that it is soon to distribute an Irix version of the Java3D runtime environment.

Sun maintains a list of ports of the Java environment at http://java.sun.com/cgi-bin/java-ports.cgi

Many of the links to ports of the Java virtual machine for other operating systems also contain information regarding the status or existence of implementations of Java3D on those platforms.
